Hexo 博客使用 LaTeX

2022/6/17 教程 Hexo LaTeX

1. 用 Codecogs 编辑 LaTeX

众所周知，Hexo 是不自带 LaTeX 的
所以，网上就有一些文章，说要改渲染引擎，要用 MathJax，要改……
但是我比较懒，所以就用了 Codecogs 的 LaTeX 渲染引擎，编辑好 LaTeX 公式，在下面选择 SVG，复制链接即可
注意复制链接后在 Markdown 文件中要用 <img> 标签，不然 Markdown 解析可能出错

2. 使用图床存储

如果 Codecogs 崩了，怎么办？

这种情况我们就要用图床存储
而一些常见的图床都不能存储 SVG 图片，我们就需要自建图床 我怎么又不懒了？

  1. 软件下载
    下载 PicGo，安装一路确认即可

  2. 创建 Github 仓库和 Token
    创建一个仓库，名称随意，用于存储图片
    Token 在这里创建，点击 Generate new token，Note 随便填，Select-scopes 选中 repo 及子项
    注意创建完 Token 不要刷新，第二次访问时 Token 就会隐藏！

  3. 配置 Picgo
    打开 Picgo，选择 图床设置/Github 图床，配置 Picgo
    仓库名填 <name>/<repo>，<name> 是你的 Github 用户名，<repo> 是你刚才创建的仓库名
    分支名填 master 或 main，具体看打开你仓库后左边显示的分支名，Github 的默认分支在这里更改
    Token 填你刚才 Github 创建的 Token

  4. 使用 Picgo
    上传：点击上传区根据提示上传
    获取链接：点击相册，找到上传的图片，点图片下最左边的按钮复制链接

3. 部署到 Netlify 加速访问

先注册一个 Netlify 账号，通过 Github 注册
注册成功后跳转到你的首页，点击 Add new site 中的 Import an existing project，点击 Github，与 Github 关联
关联 Github 后在下面会出现你 Github 中的仓库，点击你创建的图床仓库，再点击 Deploy site 上传
上传完成会跳转到项目主页，选择 Site settings，跳转后往下滑，点击 Change site name 更改图床的名称

之后就可以通过 <name>.netlify.app/<image> 访问图床了，<name>是你 Change site names 时设置的名称，<image>是你上传的图片文件名

部署到 Netlify 后访问还是很快的，SVG 图像都能约 100ms 的时间加载完成，但是 PNG/JPG 等格式最好还是用网络上常见的图床，推荐一个 https://img.gejiba.com/

4. 记录 LaTeX 公式源码

如果图床崩了，而你找不到原来的 LaTeX 公式，怎么办？

虽然这种情况不大可能出现，但我们还是要考虑到
可以把 LaTeX 公式源码复制在你 Markdown 文件中的图片后面，加上注释，这样图床崩了还能从注释中找到源码
